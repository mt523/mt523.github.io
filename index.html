Up and running with React Native for Android on Windows

The React Native framework has recently been updated to support the Android platform. Officially, OS X is the only supported operating system for development, but if you're running Windows you're not entirely out of luck. Listed below are the steps needed to get your Windows development environment up and running.

Prerequisites

If you've done any previous Android development, you should already have most of this set up.

  • Android SDK
  • Latest JDK
  • Latest version of Node.js
  • ANDROID_HOME environment variable with the correct path to your SDK

Setting up the emulator

You'll need to create an Android emulator to run the React Native project we'll create later, along with the latest SDK platform, which at the time of writing this was API 23. Simply follow the steps outlined in React Native's Android Setup guide.

Installing the React Native packages

After installing Node, we can use the Node Package Manager to install the React Native packages:

npm install react-native-cli

And that's it, you're ready to get going.

Creating a new project

Create a new directory for your new project with any name you like. From inside this directory, run the following command:

react-native init (your project name)

Modifying the generated code for Windows

All credit here goes to GitHub user "mqli" for figuring out this hack work around running the bash files. In your newly created project, follow the instructions on mqli's Gist.

Run the project

You're ready to go. Start your emulator, and run the following commands from your new project's root directory.

Start the webserver

react-native start

Run the project

react-native run-android

You should now be greeted by the Welcome to React Native screen.

For more information, visit the React Native documentation and tutorials.
